# Contributing Guidelines

Thank you for your interest in contributing to Scrim Discord Bot.

Before contributing, please read this carefully.

## Important Notice

This project is NOT open source.

All rights are reserved by the author.

Contributions do NOT grant any ownership rights over the project.

By submitting a pull request, you agree that:

* Your contribution becomes part of the project.
* The author retains full ownership of the code.
* You do not gain rights to redistribute or sell the project.

## How to Contribute

1. Fork the repository
2. Create a new branch
3. Make your changes
4. Test your changes
5. Submit a Pull Request

## Contribution Rules

* Follow the existing project structure
* Write clean and readable code
* Do not commit secrets or tokens
* Ensure new features do not break existing logic
* Keep commits clear and meaningful

## Bug Reports

When opening an issue include:

* Description of the problem
* Steps to reproduce
* Expected behavior
* Screenshots if possible

## Feature Requests

You may suggest features through issues.

Approval of feature implementation is at the sole discretion of the author.

## Code of Conduct

Be respectful and professional.
Toxic behavior will result in removal from collaboration.
